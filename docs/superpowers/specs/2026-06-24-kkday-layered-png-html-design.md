# KKday 一键安装图分层 PNG + HTML 试验设计

## 背景

当前 `text2html-image` 的复杂海报复刻流程遇到三个问题：

1. 将素材堆成透明贴纸板后，Codex 很难可靠判断每个元素在原图中的位置。
2. ChatGPT Images 2 生成的抠图素材容易出现渐变光晕、脏边和图层关系错误。
3. 复杂排版海报需要 I18N，文字空间会随语言变化，因此文字不能烘焙进图片。

本试验不再生成“素材贴纸板”，而是针对 KKday 一键安装 eSIM 图直接生成与最终画布同尺寸的透明 PNG 图层，再用 HTML 叠放这些图层，并让 HTML/CSS/SVG 承担文字和简单几何图形。

参考图尺寸已确认：`960 x 1280`。

## 目标

第一轮试验目标是验证方案 B 是否可行：

- 所有复杂图形以同尺寸透明 PNG 图层存在。
- 每个 PNG 图层尺寸均为 `960 x 1280`，透明区域保留 alpha。
- HTML 画布同样为 `960 x 1280`。
- HTML 只通过 `position:absolute` 和 `z-index` 叠放图层，不维护单个图案坐标清单。
- 所有海报级文字由 DOM 文本渲染，保留后续 I18N 能力。
- 明显的大矩形、正圆形、圆角卡片、pill、按钮底、信息条、简单线条、简单 SVG 图标由网页 CSS/SVG 负责。

## 非目标

本试验暂不做以下事情：

- 不实现通用自动切图命令。
- 不生成每个独立素材的坐标 JSON。
- 不要求手机屏幕内部所有小字第一轮全部可本地化。
- 不追求一次性覆盖所有图片类型。
- 不把复杂图形强行转成 SVG path。

## 方案选择

### 方案 A：单张透明图形层

把所有非文字图形合成一张 `graphics.png`，HTML 只在上方排文字。

优点是最简单；缺点是当文字需要夹在背景图形和前景图形之间时无法处理，手机高光、装饰线、遮挡关系容易错误。

### 方案 B：按视觉层级拆多张同尺寸透明 PNG

把复杂图形拆成若干同尺寸透明层，例如背景图形、设备图形、前景遮挡图形。HTML 文本层放在合适的 z-index 中间。

这是本试验采用的方案。它不需要单个素材坐标清单，同时能表达前后遮挡关系，并且适合后续 I18N。

### 方案 C：每个语言生成专属图形层

每个语言版本都重新生成或调整透明 PNG 图层。

这个方案最灵活，但会放大生成、校验和返工成本。只在某个语言的文字扩展显著破坏共享构图时作为兜底。

## 图层合同

所有图层均为 `960 x 1280` PNG，透明背景，坐标原点与 HTML 画布一致。

建议第一轮输出：

1. `background-art.png`
   - 右上角半透明地球。
   - 虚线航线、飞机、定位图钉、云朵。
   - 底部城市/地标线稿、山形、波浪、树木等复杂背景元素。
   - 不包含任何标题、步骤文案、说明文字、卖点文字。

2. `device-art.png`
   - 三台手机的复杂外壳、边框、阴影、高光。
   - 手机内复杂截图感图形可先保留在此层，但不应包含海报级主文案。
   - 如果手机屏幕文字后续要 I18N，应在第二轮拆成 HTML 文字或单独屏幕层。

3. `foreground-art.png`
   - 可选层。
   - 只用于必须覆盖 HTML 的复杂前景元素，例如手机玻璃高光、特殊反光、前景装饰线。
   - 第一轮如果没有明确遮挡需求，可以为空或不生成。

4. HTML/CSS/SVG 层
   - 主标题、步骤标题、步骤说明、底部提示、底部三个卖点文案。
   - 橙色 Travel eSIM pill、步骤圆形 badge、白色步骤卡片、底部信息条、卖点白卡、按钮底色、简单边框和简单图标。
   - 所有本地化文字需保留稳定 `data-i18n-key`。

## HTML 结构

第一轮 HTML 可以是一个静态页面：

```html
<main class="poster" style="width:960px;height:1280px">
  <img class="layer background-art" src="./assets/background-art.png" alt="" />
  <img class="layer device-art" src="./assets/device-art.png" alt="" />
  <section class="html-layer">
    <!-- editable DOM text and simple CSS/SVG geometry -->
  </section>
  <img class="layer foreground-art" src="./assets/foreground-art.png" alt="" />
</main>
```

图层规则：

- `.poster` 使用 `position: relative; overflow: hidden;`.
- `.layer` 使用 `position: absolute; inset: 0; width: 100%; height: 100%; object-fit: contain; pointer-events: none;`.
- CSS 几何层和 DOM 文本层通过明确 `z-index` 放在背景与前景之间。
- 页面不需要脚本。

## I18N 规则

- 文案不得烘焙进复杂 PNG 图层。
- 主标题、步骤文字、提示文字、卖点文字都应是 DOM 文本。
- DOM 文本节点需要 `data-i18n-key`。
- 多语言版本优先复用同一组 PNG 图层，只通过 HTML/CSS 调整字号、行高、换行、卡片高度和间距。
- 只有当某个语言导致共享图层不再适配时，才生成语言专属 PNG 层，例如 `background-art.ja.png` 或 `device-art.th.png`。

## PNG 生成提示词方向

给图像模型生成分层 PNG 时，应按层分别提示，而不是要求一次输出素材板。

通用要求：

- 输出透明 PNG。
- 画布必须为 `960 x 1280`。
- 图层中保留元素原始位置，不要重新排版。
- 不要生成或保留文字。
- 不要保留白底、脏边、渐变光晕污染、原图残留背景。
- 不要把明显 CSS 可重建的大矩形、正圆形、圆角卡片、pill、按钮底、信息条做进 PNG。

背景层提示应强调复杂背景图形；设备层提示应强调手机外壳和复杂截图感视觉；前景层提示只在存在遮挡/高光需求时生成。

## 验收标准

第一轮样例通过需要满足：

- `index.html` 能在本地直接打开。
- 画布尺寸为 `960 x 1280`。
- PNG 图层文件存在，尺寸均为 `960 x 1280`。
- PNG 图层有 alpha 透明通道。
- DOM 中存在主要海报文字，且不是图片文字。
- 明显大矩形、正圆形和圆角卡片由 CSS 生成。
- HTML 叠加后视觉上接近原图的主体结构。
- 无明显文字被 PNG 层错误遮挡。
- 无严重透明光晕污染白色卡片或文字区域。

## 风险与兜底

- 如果单个图形层混入文字，需要重新生成该层或用图像编辑清理文字。
- 如果背景层包含无法清理的光晕，应本地做 alpha 清理或将对应图形拆到更小的层。
- 如果文字需要穿插在手机屏幕内部，后续需要把手机屏幕内容拆成 HTML 层或屏幕专属透明层。
- 如果某语言文字扩展过大，优先调整 HTML 布局；仍无法适配时再创建该语言专属 PNG 图层。

## 第一轮实施范围

第一轮只处理用户提供的 KKday 一键安装 eSIM 参考图：

- 复制参考图到项目工作区 `source/`。
- 生成或整理 `background-art.png`、`device-art.png`、可选 `foreground-art.png`。
- 编写一个静态 HTML，将这些图层叠放，并用 HTML/CSS/SVG 复刻主要文字和简单几何。
- 用本地浏览器预览并截图对比。

后续若样例效果成立，再把流程沉淀为通用命令或 skill 规则。
